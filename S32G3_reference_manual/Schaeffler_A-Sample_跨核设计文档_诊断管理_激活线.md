## 1. 诊断中间件功能介绍

诊断管理中间件主要实现了诊断入口经过A核诊断代理功能，可路由到其他DoIP节点 或发送给M核，同时提供仲裁逻辑。

另外诊断管理中间件还提供 UDS API，以库的形式提供 UDS API 给车内诊断客户端应 用，方便车内诊断应用通过诊断接口访问车内诊断服务。

在诊断中间件的功能特性中，依赖A核与M核之间的跨核通信。

## 2. 跨核通信协议

通信介质：以太网通信、IPCF通信。

## 2.1 以太网通信

A 核 AsfDiagnosticProxy 作为 DOIP Client，M 核 CP-DM（或其它）作为 DOIP Server， 二者之间通信使用DOIP通信协议（ISO 13400-2 2012） 。

（1） Tester ：诊断仪作为DOIP Client 与 AsfDiagnosticProxy 通过以太网连接，诊断 仪源地址 0x0e80，源 IP=10.1.0.xx，目标 IP=10.1.0.79（Tx 口 IP） ，源端口=随

<!-- image -->

- 机，目标端口=13400；发送诊断请求、接收诊断响应。
- （2） AsfDiagnosticProxy ： A 核DP进程作为DOIP Server与诊断仪通过以太网连接， 作为DOIP Client 与 CP-DM 通过以太网连接；DP 转发使用源地址 0x0e80，源 IP=192.168.21.21，目标 IP=192.168.21.21（M 核 IP） ，源端口=随机，目标端口 =13400；接收诊断请求并路由、接收诊断响应并路由。
- （3） CP-DM ： M 核 DM （DOIP、 DCM、 DEM） 作为 DOIP Server 与 AsfDiagnosticProxy 通过以太网连接；接收诊断请求、处理诊断服务、发送诊断响应。

## 2.2 IPCF 通信

A 核 AsfDiagnosticInfo 作为 IPCF Server，M 核 SWC 作为 IPCF Client，M 核周期向 A 核 上报DOIP 激活线状态。

<!-- image -->

- （1） AsfDiagnosticInfo ： 作为 IPCF Server 从 M 核接收 DOIP 激活线状态， 如果 DOIP 激活则收集VIN码、EID和GID，通知AsfDiagnosticProxy。
- （2） SWC：作为IPCF Client 周期从 M 核底层获取 DOIP 激活线状态，并通知 A 核 AsfDiagnosticInfo。
