## 1. 诊断中间件功能介绍
诊断管理中间件主要实现了诊断入口经过A核诊断代理功能，可路由到其他DoIP节点或发送给M核，同时提供仲裁逻辑。
另外诊断管理中间件还提供UDS API，以库的形式提供UDS API给车内诊断客户端应用，方便车内诊断应用通过诊断接口访问车内诊断服务。
在诊断中间件的功能特性中，依赖A核与M核之间的跨核通信。
2.	跨核通信协议
通信介质：以太网通信、IPCF通信。
2.1	以太网通信
A核AsfDiagnosticProxy作为DOIP Client，M核CP-DM（或其它）作为DOIP Server，二者之间通信使用DOIP通信协议（ISO 13400-2 2012）。
 
（1）	Tester：诊断仪作为DOIP Client与AsfDiagnosticProxy通过以太网连接，诊断仪源地址0x0e80，源IP=10.1.0.xx，目标IP=10.1.0.79（Tx口IP），源端口=随机，目标端口=13400；发送诊断请求、接收诊断响应。
（2）	AsfDiagnosticProxy：A核DP进程作为DOIP Server与诊断仪通过以太网连接，作为DOIP Client与CP-DM通过以太网连接；DP转发使用源地址0x0e80，源IP=192.168.21.21，目标IP=192.168.21.21（M核IP），源端口=随机，目标端口=13400；接收诊断请求并路由、接收诊断响应并路由。
（3）	CP-DM：M核DM（DOIP、DCM、DEM）作为DOIP Server与AsfDiagnosticProxy通过以太网连接；接收诊断请求、处理诊断服务、发送诊断响应。
2.2	IPCF通信
A核AsfDiagnosticInfo作为IPCF Client Server，M核SWC作为IPCF Server Client，M核周期向A核上报DOIP激活线状态。
 
（1）	AsfDiagnosticInfo：作为IPCF Client Server从M核接收DOIP激活线状态，如果DOIP激活则收集VIN码、EID和GID，通知AsfDiagnosticProxy。
（2）	SWC：作为IPCF Server Client周期从M核底层获取DOIP激活线状态，并通知A核AsfDiagnosticInfo。
